# `model/pretrained/train.py` 캐시 파일명 변경 안내

_작성일: 2026-05-10 | 대상: SafeSignal 팀 전원 (특히 사전학습/파인튜닝 담당)_

## 요약

사전학습 캐시 파일명이 전처리 파라미터를 자동으로 박는 형식으로 바뀌었습니다.
**원칙: `--cache_path`를 직접 지정하지 말고 기본값을 그대로 쓰세요.**

## 바뀐 점

- 캐시 파일명 형식: `dataset_cache_e{envs}_w{window_size}_s{stride}[_tail][_ps].npz`
- 예: 기본값으로 돌리면 `dataset_cache_e12_w300_s300_tail_ps.npz`
- `train.py` 상단의 `CACHE_WINDOW_SIZE / CACHE_STRIDE / CACHE_TAIL_WINDOW / CACHE_PAD_SHORT` 중 하나라도 바뀌면 파일명이 자동으로 달라져서 **새 캐시가 만들어지고, 동일 파라미터의 기존 캐시는 자동 재활용**됩니다. 이제 `--rebuild_cache`를 손으로 붙일 필요가 없습니다.

## 왜 `--cache_path`를 직접 주지 말라고 하는가

`--cache_path foo.npz`처럼 경로를 박아 넣으면, 안의 데이터가 어떤 파라미터로 만들어졌는지 파일명이 더 이상 보장하지 않습니다. 그래서 다음과 같은 사고가 가능해집니다.

1. A가 `--cache_path my.npz`로 캐시를 만든다 (그 시점 stride=300).
2. 나중에 누군가가 `CACHE_STRIDE=100`으로 바꾼다.
3. B가 `--cache_path my.npz`로 학습을 돌린다 → 파일명은 그대로니 **재빌드 없이 stride=300짜리 옛 캐시로 학습**된다. 코드와 데이터의 파라미터가 어긋난 채 결과만 나옴 — 디버깅이 매우 어렵습니다.

기본 경로(`--cache_path` 생략)를 쓰면 파일명에 파라미터가 박혀 있어 위 상황에서 자동으로 다른 파일을 보게 되므로 사고가 원천 차단됩니다.

## 그래서 어떻게 쓰면 되는가

| 상황                                                                 | 어떻게                                                                                                       |
|----------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------|
| 평소 학습 (기본)                                                     | `python -m model.pretrained.train --epochs 30` — 그냥 돌리세요. 기존 캐시가 있으면 알아서 재사용.              |
| 전처리 파라미터(window/stride/tail/ps)를 바꾸고 싶을 때              | `train.py` 상단의 `CACHE_*` 상수를 수정하고 그냥 다시 돌리세요. 새 파일명으로 자동 재빌드됩니다.               |
| 같은 파라미터로 강제로 캐시를 다시 만들고 싶을 때 (디스크 손상 의심 등) | `--rebuild_cache` 플래그만 추가. 경로는 여전히 기본값으로.                                                     |
| `--envs 1`처럼 환경 부분집합으로 빠른 검증                           | `--envs 1`만 주세요. 캐시 파일명에 envs가 포함되어 자동 분리됩니다(`dataset_cache_e1_...`).                    |

## `--cache_path`를 써도 되는(드문) 경우

- RTX4060 서버에서 만든 캐시 `.npz`를 노트북으로 복사한 뒤 다른 디렉터리에서 그대로 가리키고 싶을 때
- 일회성 실험을 위해 캐시를 별도 이름으로 보존하고 싶을 때 (예: `--cache_path tmp_exp1.npz`)
- 위 두 경우 모두 **그 파일이 어떤 파라미터로 만들어졌는지 본인이 책임지고 알고 있어야** 합니다. 이름에 파라미터를 명시해서 저장하는 걸 권장합니다 (예: `--cache_path foo_w300_s100.npz`).

## 한 줄 요약

> 그냥 `python -m model.pretrained.train ...`로 돌리고, 파라미터를 바꾸고 싶으면 `train.py`의 `CACHE_*` 상수만 수정. `--cache_path`는 캐시를 직접 옮기거나 보관할 때만 사용.
